# API

## Общие замечания

- Базовый префикс модуля: `/api/equipment`
- Идентификатор в URL — это пользовательский `visibleId`, а не внутренний `equipment.id`
- Для создания и редактирования оборудования требуется право на редактирование оборудования
- Для загрузки файлов требуется право на управление файлами оборудования
- Ошибки валидации и бизнес-ошибки возвращаются как `400` или `409`

## GET /api/equipment

Получение списка оборудования.

### Назначение

Возвращает реестр оборудования для таблицы списка.

### Параметры

Нет.

### Ответ

Массив объектов списка:

- `id`
- `visibleId`
- `name`
- `inventoryNumber`
- `serialNumber`
- `manufacturer`
- `model`
- `status`
- `statusLabel`

### Ошибки

Специальные ошибки модулем не формируются.

## GET /api/equipment/create-options

Получение справочников для формы создания и редактирования.

### Назначение

Возвращает набор данных, необходимых для построения формы оборудования.

### Ответ

Объект:

- `nextVisibleId`
- `manufacturers`
- `models`
- `countries`
- `sections`
- `employees`
- `statuses`

### Ошибки

Специальные ошибки модулем не формируются.

## GET /api/equipment/:visibleId

Получение карточки оборудования.

### Параметры

- `visibleId` — целый положительный номер оборудования в URL

### Ответ

Полная карточка оборудования:

- идентификаторы и названия;
- модель и производитель;
- страна;
- даты;
- статус и человекочитаемая подпись статуса;
- местоположение;
- ответственный сотрудник;
- текстовые поля.

### Ошибки

- `404` — оборудование не найдено

## GET /api/equipment/:visibleId/history

Получение истории изменений оборудования.

### Назначение

Возвращает строки аудита для карточки оборудования.

### Ответ

Массив записей:

- `id`
- `action`
- `createdAt`
- `fieldName`
- `oldValue`
- `newValue`
- `timeZone`
- `user`

### Ошибки

- `404` — оборудование не найдено

## POST /api/equipment

Создание оборудования.

### Назначение

Создаёт новую запись оборудования и возвращает полную карточку.

### Тело запроса

- `visibleId?`
- `name`
- `modelId`
- `specifications?`
- `serialNumber?`
- `inventoryNumber`
- `countryId?`
- `manufactureYear?`
- `commissioningDate?`
- `issueDate`
- `sectionId`
- `responsibleEmployeeId`
- `status`
- `operationText?`
- `notes?`

### Ответ

Полная карточка оборудования в том же формате, что `GET /api/equipment/:visibleId`.

### Ошибки

- `400` — некорректные данные формы
- `400` — модель, участок, страна или ответственный не существуют
- `400` — ответственный сотрудник неактивен
- `409` — `visibleId` уже занят
- `409` — `inventoryNumber` уже занят

## PUT /api/equipment/:visibleId

Обновление оборудования.

### Назначение

Изменяет существующее оборудование и возвращает обновлённую карточку.

### Параметры

- `visibleId` — текущий пользовательский номер оборудования в URL

### Тело запроса

Такое же, как у `POST /api/equipment`.

### Ответ

Полная карточка оборудования.

### Ошибки

- `404` — оборудование не найдено
- `400` — некорректные данные формы
- `400` — модель, участок, страна или ответственный не существуют
- `400` — ответственный сотрудник неактивен
- `409` — новый `visibleId` уже занят
- `409` — `inventoryNumber` уже занят

## GET /api/equipment/:visibleId/files

Получение списка файлов оборудования.

### Назначение

Возвращает файлы, привязанные к оборудованию как к владельцу в storage-модуле.

### Ответ

Массив объектов файла storage-модуля.

Каждый объект содержит метаданные файла, используемые фронтендом для отображения списка и скачивания:

- идентификатор файла;
- имя файла;
- тип документа;
- размер;
- дату создания;
- автора или источник загрузки;
- служебные поля storage-модуля, необходимые для работы с файлом.

### Ошибки

- `404` — оборудование не найдено

## POST /api/equipment/:visibleId/files

Загрузка файла оборудования.

### Назначение

Загружает файл в storage-модуль и пишет аудит загрузки в модуле `EQUIPMENT`.

### Тело запроса

`multipart/form-data`:

- `file`
- `documentType`

### Ответ

Созданная запись файла из storage-модуля.

Ответ содержит метаданные загруженного файла и данные, необходимые для его последующего отображения в списке файлов оборудования.

### Ограничения

- размер файла ограничен `MAX_FILE_SIZE_BYTES`
- требуется право `assertCanManageFiles()`

### Ошибки

- `404` — оборудование не найдено
- ошибки валидации storage-модуля
