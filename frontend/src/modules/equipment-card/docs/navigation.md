# Навигация

## Источник маршрута

Модуль работает поверх hash-router приложения.

Он не парсит URL браузера напрямую, а опирается на:

- `initialTab`, переданный сверху;
- `returnTo`, переданный сверху;
- `window.location.hash` для синхронизации после пользовательских действий.

## Параметр `tab`

Параметр `tab` хранит активную вкладку карточки.

Допустимые значения:

- `details`
- `documents`
- `events`
- `maintenance-settings`
- `history`

Парсинг значения вынесен в `parseEquipmentViewTab()`.

Если значение отсутствует или некорректно, используется:

- `details`

## Параметр `returnTo`

Параметр `returnTo` хранит маршрут возврата в предыдущий экран, обычно в список оборудования.

Сам модуль карточки не валидирует `returnTo`, а только сохраняет его при:

- переключении вкладок;
- переходе в редактирование.

Общая утилита `getSafeReturnTo()` находится в `shared/lib/hash-navigation.ts`, но внутри `EquipmentCardView` она напрямую не вызывается.

## Алгоритм восстановления вкладки

Восстановление вкладки состоит из двух частей:

1. Внешний слой страницы определяет `initialTab`.
2. `EquipmentCardView` инициализирует `activeTab` этим значением.

Дополнительно внутри `EquipmentCardView` есть эффект:

- при изменении `initialTab` локальный `activeTab` принудительно обновляется.

Это позволяет переиспользовать один экземпляр компонента и синхронизировать его с новым hash-маршрутом.

## Переключение вкладки

При смене вкладки `EquipmentCardView`:

1. Проверяет, не выбрана ли уже активная вкладка.
2. Обновляет локальный `activeTab`.
3. Вызывает внешний `onTabChange`, если он передан.
4. Строит новый hash через `buildHashRoute()`.
5. Обновляет `window.location.hash`, если значение действительно изменилось.

Для вкладки `details` параметр `tab` из hash удаляется:

- `tab: null`

Это делает `details` значением по умолчанию.

## Переход в редактирование

Кнопка `Редактировать` строит маршрут:

- `#/equipment/{visibleId}/edit`

К нему добавляются:

- `returnTo`
- `tab`, если активная вкладка не `details`

Маршрут формируется через `buildHashRoute()`.

## View Transition API

Переход в редактирование не меняет `window.location.hash` напрямую из обработчика ссылки.

Используется утилита `navigateWithViewTransition()`:

- если новый hash совпадает с текущим, переход не выполняется;
- если `document.startViewTransition` недоступен, hash меняется напрямую;
- если API доступен, переход оборачивается в `startViewTransition`;
- если `startViewTransition` выбрасывает исключение, выполняется fallback на прямую смену hash.

## Поведение при отсутствии параметров

Если `initialTab` не передан:

- активной вкладкой становится `details`

Если `returnTo` передан пустым или некорректным на уровне страницы:

- карточка всё равно только переносит это значение дальше;
- нормализация должна выполняться внешним слоем.
