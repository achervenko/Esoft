# Фотографии

## Источник данных

Фотографии загружаются через:

- `getEquipmentFiles(visibleId)`

API возвращает все файлы оборудования, а не только фотографии.

## Фильтрация

Фильтрация выполняется внутри `useEquipmentPhotos`.

Во внешний компонент возвращаются только файлы, у которых:

- `documentType === "equipment_photo"`

Отдельного типа `EquipmentPhoto` в коде нет. Модуль использует:

- `EquipmentFile`

как общий тип файла и рассматривает фотографии как его подмножество.

## Когда выполняется запрос

Запрос фотографий выполняется только когда:

- активна вкладка `details`

Это контролируется параметром:

- `enabled: activeTab === "details"`

Если `enabled === false`, хук не делает запрос.

## Повторная загрузка

При повторном открытии вкладки `details` хук снова выполняет запрос.

Текущая реализация не кеширует результат между последовательностями:

- `details -> другая вкладка -> details`

## Состояния загрузки

`useEquipmentPhotos` возвращает:

- `photos`
- `isLoading`
- `error`

Внутри хука:

- `isLoading` ставится в `true`, если текущий `visibleId` ещё не совпадает с `filesVisibleIdRef.current`
- `error` очищается перед новым запросом
- при ошибке для нового оборудования старые фотографии удаляются

## Защита от устаревшего результата

Хук использует локальный флаг:

- `isMounted`

Он предотвращает обновление состояния после размонтирования или после завершения эффекта.

Дополнительно хук хранит:

- `filesVisibleId`
- `filesVisibleIdRef`

Это позволяет не показывать файлы от предыдущего оборудования, пока новое состояние ещё не синхронизировано.

## Галерея

Компонент:

- `EquipmentPhotoGallery`

выполняет три задачи:

- сортирует список фотографий;
- управляет активной фотографией;
- открывает полноэкранный preview.

### Сортировка

Фотографии сортируются так:

1. `isPrimary === true` идёт первой
2. при одинаковом `isPrimary` выше идут файлы с большим `id`

### Активная фотография

Галерея хранит:

- `activeIndex`

Активная фотография вычисляется как:

- `sortedPhotos[activeIndex] ?? null`

При изменении состава фотографий через `photoSetKey` галерея:

- сбрасывает `activeIndex` в `0`
- закрывает preview

### Плейсхолдеры

Если активной фотографии нет, показывается placeholder:

- `Загрузка фото...`, если `isLoading === true`
- текст ошибки, если `error !== null`
- `Фото не загружено` в остальных случаях

### Переключение

Если фотографий больше одной:

- показываются кнопки `Предыдущее фото` и `Следующее фото`
- показывается счётчик вида `N / M`

Переключение циклическое:

- назад с первого элемента переводит на последний
- вперёд с последнего переводит на первый

## Полноэкранный просмотр

Полноэкранный просмотр выполняет:

- `ImagePreviewModal`

### Открытие

Preview открывается по клику на активную фотографию.

### Закрытие

Preview закрывается:

- по кнопке `Закрыть`
- по клику на backdrop
- по `Escape`

### Клавиатурное управление

Модалка обрабатывает:

- `Escape`
- `Tab`
- `Shift+Tab`

### Focus trap

`ImagePreviewModal` собирает focusable-элементы через `getFocusableElements()` и замыкает фокус внутри диалога.

Если focusable-элементов нет:

- фокус остаётся на самом диалоге

### Восстановление фокуса

При открытии модалка сохраняет предыдущий `document.activeElement`.

При закрытии:

- возвращает фокус на этот элемент, если он `HTMLElement`

### Блокировка прокрутки страницы

При открытии:

- `document.body.style.overflow = "hidden"`

При размонтировании:

- исходное значение `overflow` восстанавливается
