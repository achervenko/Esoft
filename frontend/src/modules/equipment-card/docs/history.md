# История изменений

## Структура записи

История работает с типом:

- `EquipmentHistoryItem`

Поля, которые реально используются во frontend:

- `id`
- `action`
- `createdAt`
- `fieldName`
- `oldValue`
- `newValue`
- `user`
- `timeZone`

Поле `timeZone` приходит из API, но текущее форматирование даты в `EquipmentHistoryView` использует константу:

- `BUSINESS_TIME_ZONE = "Europe/Moscow"`

и подпись:

- `МСК`

## Группировка изменений

`EquipmentHistoryView` не показывает историю как плоский список.

Сначала записи группируются через `groupHistoryItems()`.

Две записи попадают в одну группу, если одновременно совпадают:

- `action`
- `user`
- разница между `createdAt` не больше `15_000` мс

Это окно задаётся константой:

- `HISTORY_GROUP_WINDOW_MS = 15_000`

## Группировка файлов

Файловыми считаются действия:

- `FILE_UPLOAD`
- `FILE_DELETE`

Для них используется отдельное представление:

- иконка загрузки или удаления;
- список имён файлов внутри группы.

Текущая реализация не скрывает дополнительные элементы группы: если в группе несколько файловых записей, отображаются все значения из `group.items`.

## Отображение обычных изменений

Для действий, отличных от файловых, показывается таблица с колонками:

- `Поле`
- `Старое значение`
- `Новое значение`

Размер группы отображается как:

- `1 поле`
- `2 поля`
- `5 полей`

## Отображение действий

Для известных action используются человекочитаемые подписи:

- `CREATE` -> `Создание`
- `UPDATE` -> `Изменение`
- `FILE_UPLOAD` -> `Документ загружен`
- `FILE_DELETE` -> `Документ удалён`

Если action неизвестен:

- показывается исходное значение action

## Форматирование значений

Пустыми считаются значения:

- `null`
- пустая строка
- строка из пробелов
- строка `не указано` в любом регистре для `ru-RU`

Во всех этих случаях выводится:

- `Не указано`

## Форматирование дат

Дата форматируется через `Intl.DateTimeFormat("ru-RU", ...)` с параметрами:

- `timeZone: BUSINESS_TIME_ZONE`
- `day`, `month`, `year`, `hour`, `minute`

После даты добавляется суффикс:

- `МСК`

Если `createdAt` не парсится:

- показывается `Дата не указана`

## Ошибки и пустое состояние

Если история не загружена и массив пуст:

- `EquipmentHistoryView` показывает empty state

Если история загружается или есть ошибка:

- это обрабатывается уровнем `EquipmentHistoryTab`
