# Вкладки

## Общая модель

В модуле есть пять вкладок:

- `details`
- `documents`
- `events`
- `maintenance-settings`
- `history`

Список вкладок фиксирован внутри `EquipmentCardTabs`.

## details

Назначение:

- показать основную карточку оборудования.

Источник данных:

- `equipment: EquipmentCard`
- `photos` из `useEquipmentPhotos`

Условия отображения:

- всегда доступна;
- является вкладкой по умолчанию.

Доступные действия:

- просмотр всех основных полей;
- просмотр текстовых блоков;
- просмотр фотографий;
- открытие полноэкранного preview;
- переход в редактирование при `canEdit === true`.

Особенности:

- только на этой вкладке включается `useEquipmentPhotos`;
- hash-параметр `tab` для неё не сохраняется.

## documents

Назначение:

- показать вложенный модуль документов оборудования.

Источник данных:

- `visibleId`

Условия отображения:

- всегда может быть открыта;
- содержимое монтируется только когда вкладка активна.

Доступные действия:

- зависят от внутренней логики `EquipmentDocumentsPanel`.

Права доступа:

- сама карточка не скрывает вкладку по правам;
- edit-кнопка в шапке всё ещё видна при `canEdit === true`.

Особенности:

- `EquipmentDocumentsPanel` получает `mode="view"`.

## events

Назначение:

- показать вложенный модуль событий оборудования.

Источник данных:

- `visibleId`
- `equipment.status`

Условия отображения:

- содержимое монтируется только в активном состоянии.

Доступные действия:

- зависят от `EquipmentEventsPanel`.

Права доступа:

- флаг `canManageEquipmentEvents` пробрасывается как `canManageEvents`;
- вкладка доступна и без права управления, но внутренний модуль может ограничивать действия.

Особенности:

- кнопка `Редактировать` в шапке карточки для этой вкладки не показывается.

## maintenance-settings

Назначение:

- показать вложенный модуль настроек обслуживания.

Источник данных:

- `visibleId`

Права доступа:

- `canManageMaintenanceSettings` пробрасывается как `canManage`.

Особенности:

- вкладка остаётся видимой даже без права управления;
- ограничения по действиям реализуются во вложенной панели.

## history

Назначение:

- показать историю изменений оборудования.

Источник данных:

- `history: EquipmentHistoryItem[]`
- `historyError`
- `isHistoryLoading`

Условия отображения:

- рендерится только когда активна.

Доступные действия:

- просмотр истории.

Права доступа:

- отдельного флага внутри карточки нет.

Особенности:

- кнопка `Редактировать` на этой вкладке не отображается.
